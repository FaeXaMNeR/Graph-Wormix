# Worms Math Revolution

Сетевая артиллерийская игра на Python + pygame. Игроки стреляют по друг другу траекториями, заданными математическими функциями `y = f(x)`.

## Установка

```
pip install pygame
```

Сервер (`server.py`) требует только стандартную библиотеку Python.

## Запуск

### Быстрый тест на одном компьютере

Запустите сервер и несколько клиентов на одном ПК:

```
# Терминал 1 — сервер
python server.py --local

# Терминал 2 — игрок 1
python client.py 127.0.0.1 5555 Игрок1

# Терминал 3 — игрок 2
python client.py 127.0.0.1 5555 Игрок2
```

Флаг `--local` привязывает сервер к `127.0.0.1` — только локальные подключения.

### Локальная игра (в одной сети)

1. На одном компьютере в локальной сети запустите сервер:
```
python server.py --port 5555 --players 2
```
`--players N` — игра начнётся автоматически, когда подключится N игроков (от 2 до 6).

2. Каждый игрок запускает клиент, указав IP-адрес сервера:
```
python client.py 192.168.1.50 5555 Имя
```

### Сетевая игра через интернет (публичный IP)

#### Настройка сервера на виртуальной машине

1. **Запустите сервер** на виртуальной машине:
```
python server.py --port 5555 --players 2
```

2. **Узнайте публичный IP** виртуальной машины (сервер покажет его автоматически).

3. **Настройте файрвол** на виртуальной машине:
   - **Windows**: Разрешите входящие соединения для порта TCP 5555
     ```powershell
     netsh advfirewall firewall add rule name="Wormix" dir=in action=allow protocol=TCP localport=5555
     ```
   - **Linux**: 
     ```bash
     sudo ufw allow 5555/tcp
     # или
     sudo iptables -A INPUT -p tcp --dport 5555 -j ACCEPT
     ```

4. **Настройте проброс портов** на роутере (если VM за NAT):
   - Войдите в веб-интерфейс роутера (обычно 192.168.1.1)
   - Найдите раздел "Port Forwarding" или "Проброс портов"
   - Добавьте правило:
     - Внешний порт: 5555
     - Внутренний IP: IP виртуальной машины
     - Внутренний порт: 5555
     - Протокол: TCP

5. **Поделитесь IP** с друзьями — они подключятся:
```
python client.py <ПУБЛИЧНЫЙ_IP> 5555 Имя
```

#### Дополнительные опции сервера

```
python server.py --help

# Примеры:
python server.py --local                  # быстрый тест на одном ПК
python server.py --port 7777              # другой порт
python server.py --turn-timeout 30        # 30 секунд на ход (по умолчанию 180)
python server.py --players 4              # ждать 4 игрока
```

## Управление

- **Кликните в текстовое поле** внизу экрана и введите функцию `y = f(x)`
- **Enter** — ввод текста; если текст — `move dx dy`, выполняется перемещение, иначе выстрел
- **Кнопка "Огонь!"** — выстрел (только в свой ход)
- **Кнопка "Завершить ход"** — завершить ход после перемещения (можно сначала переместиться, потом выстрелить)
- **Кнопка ←/→** — flip оси X (стрелка меняет направление)
- **`move dx dy`** — перемещение (например `move 1 -2`), `dx² + dy² < 8`; после ввода можно кликнуть по карте для выбора точки
- **Координаты курсора** в локальной СК отображаются в правом нижнем углу
- **CAM** — переключение режима камеры (свободное перемещение и вращение взгляда)
- **CAM-режим**: колёсико мыши — панорамирование; стрелки на клавиатуре — панорамирование в экранных координатах

### Разрешённые функции
`sin cos tan asin acos atan sinh cosh tanh exp log log10 sqrt abs floor ceil pow`

Константы: `pi`, `e`. Переменная: `x`.

Примеры: `sin(x)/4`, `x*0.3`, `x**2/500 - 60`, `-5*x`

## Правила

- Поле **1600×900** с процедурными препятствиями (квадраты, круги, прямоугольники, 30% заполнение).
- У каждого игрока **5 HP** и **своя система координат**, повёрнутая на случайный угол.
- Углы обновляются каждый новый раунд (когда все игроки сделали ход).
- В свой ход можно **переместиться** (ввести `move dx dy`) **и/или выстрелить** (ввести функцию и нажать Enter/Огонь!). Оба действия выполняются за один ход.
- Вводится функция `y = f(x)` в **локальной** (повёрнутой) системе координат — это траектория снаряда.
- Шум траектории растёт линейно с расстоянием, отклонения **перпендикулярны касательной** траектории.
- Снаряд разрушает препятствия при попадании (круглая область взрыва).
- Игра заканчивается, когда остаётся 0 или 1 игрок в живых.
- По окончании — голосование за реванш.
- При завершании игры и отключении всех игроков — автоматический рестарт.

### Спавн
- Каждый игрок выбирает стартовую позицию кликом по полю.
- Если позиция на препятствии или слишком близко к другому — **все** расставляются заново.
- После 3 неудач подряд — авто-расстановка.

## Бонусы

Появляются на чётных раундах начиная со 2-го. Один бонус на карте, один тип не повторяется 2+ раз подряд.

| Бонус | Эффект |
|-------|--------|
| **damage** (DMG) | Следующий выстрел наносит 2 HP (сохраняется при ходе) |
| **shield** (SHD) | Блокирует одно попадание до следующего хода |
| **double_shot** (x2) | Два выстрела подряд в одном ходу |
| **angle_reset** (ANG) | В следующем ходу оси ставятся в положение 0° |

Подбираются **выстрелом** (попадание в бонус) или **движением** (проход через бонус).

Отображаются как пульсирующий фиолетовый квадрат 16×16 px. У игрока с бонусом — индикатор рядом с HP-баром.

## Режим камеры (CAM)

Нажмите кнопку **CAM** для включения/выключения свободного обзора:

- Игрок всегда位于(в центре вращения), камера показывает окрестность
- **Колёсико мыши** / **стрелки на клавиатуре** — панорамирование
- При переключении хода камера автоматически возвращается к игроку
- Сетка и оси отображаются в локальной СК игрока
- Полуплоскость (сторона выстрела) показывается жёлтым полупрозрачным.overlay
- Препятствия корректно отображаются при вращении (полигональная отрисовка)
- **Зеркальное отражение графика**: при вводе функции одновременно отображается её зеркальная копия (пунктир красного цвета), показывающая траекторию при.flip
